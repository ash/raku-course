title: Traits and pragmas
---

{% include menu.html %}

Two features let you influence how code is **compiled**, before it ever runs. A _trait_ is a modifier attached to a declaration — a variable or a routine — written with `is`. A _pragma_ is a module-like directive, switched on with `use` (or off with `no`), that changes a rule for the rest of the lexical scope.

You have already used a trait such as `is default` in passing. This section looks at traits as a feature in their own right, shows how to write your own, and introduces pragmas.
