Which Pod directive writes a **top-level heading**?

{:.quiz}
1 | `=head1`
0 | `=title`
0 | `=h1`
0 | `=heading`

{% include quiz.html %}

<div class="extended-explanation">

`=head1` is a top-level heading; `=head2`, `=head3`, and so on are deeper levels. The number is the heading level, much like in other markup languages.

</div>
