Solution: Compare say and put

Code

Here is one of the possible solutions:

my Int $i = 42;
say $i;
put $i;

my Rat $r = 3/4;
say $r;
put $r;

my Num $n = 3e4;
say $n;
put $n;

my Str $s = 'Raku';
say $s;
put $s;

my @a = <this is an array>;
# say @a.WHAT;
say @a;
put @a;

my List $l = <this is a list>;
# say $l.WHAT;
say $l;
put $l;

my %h = A => 'alpha', B => 'beta';
say %h;
put %h;

For better confidence, you can also print the type of the variable, e.g., as shown for arrays and lists, to make sure you have created a variable of the desired type.

Comments

By examining the output of the program, you can clearly see that for the given examples, there is no difference in the output for simple data types such as numbers and strings. For aggregate data types, say produces a bit more â€™noisyâ€™ output comparing to put. On the other side, for hashes, put prints it as a table comparing to a single line that say does.

The difference between the output format is determined by how the Str and gist methods are implemented for the type in hand. We will talk more about this later in the course.
